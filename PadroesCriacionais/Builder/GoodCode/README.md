## GoodCode

Para abstrair construtores muito grandes, podemos utilizar o 
patter Build é interessante pensar também que poderiamos não
utilizar esse pattern, mas essa é pelo menos uma das alternativas
tome cuidado para não adicionar complexidade demais no seu sistema.

O método build é commumente utilizado com o padrão de Fluent Interface
que básicamente faz o retorno da própria classe e possibilida que 
façamos várias chamadas encadedas de métodos.
